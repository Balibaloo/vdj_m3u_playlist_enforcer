# Installation

This project is packaged the same way as `music-file-playlist-online-sync`: a tarball is generated by `scripts/package.sh`, then the binary, example config, and systemd unit can be copied into place on the target machine.

## Build and package

```sh
chmod +x scripts/package.sh
scripts/package.sh
```

The script produces a tarball under `packaging/dist` (e.g. `vdj-m3u-playlist-enforcer-YYYYMMDDHHMMSS.tar.gz`).

## Install on the target machine

Run as root:

```sh
# Extract generated package (adjust name)
tar xzf /path/to/vdj-m3u-playlist-enforcer-YYYYMMDDHHMMSS.tar.gz -C /opt/

# Install the binary
install -m 0755 /opt/vdj-m3u-playlist-enforcer-YYYYMMDDHHMMSS/vdj-m3u-playlist-enforcer /usr/bin/vdj-m3u-playlist-enforcer

# Copy the example config and edit it
install -Dm644 /opt/vdj-m3u-playlist-enforcer-YYYYMMDDHHMMSS/example-config.toml /etc/vdj-m3u-playlist-enforcer/config.toml
# Edit /etc/vdj-m3u-playlist-enforcer/config.toml to set `root_folder`

# Install the systemd unit and enable the service
install -Dm644 /opt/vdj-m3u-playlist-enforcer-YYYYMMDDHHMMSS/systemd/vdj-m3u-playlist-enforcer.service /etc/systemd/system/vdj-m3u-playlist-enforcer.service
install -Dm644 /opt/vdj-m3u-playlist-enforcer-YYYYMMDDHHMMSS/systemd/vdj-m3u-playlist-enforcer-scan.service /etc/systemd/system/vdj-m3u-playlist-enforcer-scan.service
install -Dm644 /opt/vdj-m3u-playlist-enforcer-YYYYMMDDHHMMSS/systemd/vdj-m3u-playlist-enforcer-scan.timer /etc/systemd/system/vdj-m3u-playlist-enforcer-scan.timer
systemctl daemon-reload
systemctl enable --now vdj-m3u-playlist-enforcer.service
systemctl enable --now vdj-m3u-playlist-enforcer-scan.timer
```

## Prepare the runtime user and directories

Before starting the service, create the dedicated user and paths that the unit expects:

```sh
sudo useradd --system --no-create-home --shell /usr/bin/nologin vdj-m3u-playlist-enforcer
sudo mkdir -p /etc/vdj-m3u-playlist-enforcer /var/lib/vdj-m3u-playlist-enforcer /var/log/vdj-m3u-playlist-enforcer
sudo chown -R vdj-m3u-playlist-enforcer:vdj-m3u-playlist-enforcer /etc/vdj-m3u-playlist-enforcer /var/lib/vdj-m3u-playlist-enforcer /var/log/vdj-m3u-playlist-enforcer
```

The service runs as the `vdj-m3u-playlist-enforcer` user and monitors the configured `root_folder`. It automatically restarts on failure and starts on boot once enabled.

The scan timer fires hourly (and soon after boot) so the descriptor converter can re-run even if the watcher unit fails; enable `vdj-m3u-playlist-enforcer-scan.timer` along with the watcher service to keep everything in sync.

## Configuration

Edit `/etc/vdj-m3u-playlist-enforcer/config.toml` and set the `root_folder` to the directory that contains your VirtualDJ `.vdjfolder` playlists. Make sure the service user has read/write access to that directory (for example, via ACLs).

The enforcer will refuse to start (it logs "Configuration ... missing" and exits) if `/etc/vdj-m3u-playlist-enforcer/config.toml` is absent, so the service effectively skips running until you create that file.
